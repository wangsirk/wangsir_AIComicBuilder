# v0.3 Features Design: Markdown Import, Episode Continuation, Video Merge

## Overview

Three new features for AIComicBuilder v0.3:

1. **Markdown file import** — support `.md` files in script upload pipeline
2. **Episode continuation** ("续上集") — use previous episode's last frame as current episode's first frame
3. **Episode video merge** — combine multiple episode videos into one downloadable/playable video

---

## Feature 1: Markdown File Support

### Scope

Add `.md` / `.markdown` as accepted file types in the import pipeline, treated identically to `.txt`.

### Changes

**`src/lib/import-utils.ts`** — `extractTextFromFile()`:
- Add `.md` and `.markdown` to the switch/if block alongside `.txt`
- Read buffer as UTF-8 text, no stripping of markdown syntax (raw text passed to AI)

**`src/components/editor/upload-script-dialog.tsx`**:
- Add `.md` to the accepted file extensions in the dropzone config

**`src/app/[locale]/project/[id]/import/page.tsx`** (if applicable):
- Add `.md` to accepted file types in the full import pipeline UI

### No changes needed

- Chunking logic (`chunkText`) — works on plain text, markdown is plain text
- AI prompts — already handle arbitrary text input
- API routes — file type check happens in `extractTextFromFile()`

## Feature 2: Episode Continuation ("续上集")

### User Story

When generating storyboard frames for episode N (N > 1), the user can check "续上集" to automatically copy the last frame of episode N-1 as the first frame of episode N's first shot, ensuring visual continuity across episodes.

### UI Changes

**`src/app/[locale]/project/[id]/episodes/[episodeId]/storyboard/page.tsx`**:
- Add a checkbox "续上集" next to the "生成帧" (generate frames) button
- Only visible/enabled when current episode's sequence > 1
- Disabled state with tooltip when sequence === 1

### API Changes

**`POST /api/projects/[id]/generate`** — `handleBatchFrameGenerate`:
- Accept new parameter: `continueFromPrev: boolean`
- When `true`:
  1. Use the existing `episodeId` parameter to query the current episode's sequence number
  3. Get the last shot (by sequence) of that episode
  4. Validate that `lastFrame` exists on that shot; if not, return error: "上一集尚未生成帧，无法续接"
  5. Copy the `lastFrame` file to a new path for the current episode's first shot
  6. Update the first shot's `firstFrame` field in DB
  7. Skip firstFrame generation for the first shot; proceed to lastFrame generation
  8. All subsequent shots follow existing sequential generation logic

### Data Flow

```
Episode N-1, Last Shot → lastFrame (file path)
        ↓ (file copy)
Episode N, Shot 1 → firstFrame (new file path)
        ↓ (skip firstFrame generation)
Episode N, Shot 1 → generate lastFrame (using copied firstFrame as reference)
        ↓ (normal flow continues)
Episode N, Shot 2 → firstFrame generated using Shot 1's lastFrame
        ...
```

### Edge Cases

- Episode sequence = 1: checkbox hidden/disabled
- Previous episode has no shots: return error with message
- Previous episode's last shot has no `lastFrame`: return error with message
- User unchecks "续上集": normal generation, no cross-episode reference
- User re-generates episode N-1 frames after using "续上集" for episode N: the copied frame in episode N becomes stale — this is the user's responsibility; no auto-sync

## Feature 3: Episode Video Merge

### User Story

After composing videos for multiple episodes, the user can select episodes and merge their final videos into a single video for preview and download.

### UI Changes

**`src/app/[locale]/project/[id]/episodes/page.tsx`**:
- Add "合并视频" button in the page header/toolbar area
- Clicking it toggles "selection mode":
  - Each episode card shows a checkbox overlay
  - Only episodes with `finalVideoUrl` are selectable (others grayed out)
  - A floating action bar appears at bottom with: selected count, "取消", "确认合并"
- After merge completes:
  - Show video player modal with the merged video
  - Provide download button

**`src/components/editor/episode-card.tsx`**:
- Accept `selectionMode: boolean` and `selected: boolean` and `selectable: boolean` props
- Show checkbox when in selection mode
- Visual indication for non-selectable episodes (no final video)

### API

**`POST /api/projects/[id]/merge-episodes`** (new route):
- Request body: `{ episodeIds: string[] }`
- Prerequisite: Episode-level video assembly already exists — `handleVideoAssembleSync` in `generate/route.ts` populates `episodes.finalVideoUrl` when called with an `episodeId`. No new work needed for this.
- Validation:
  - All episode IDs belong to this project
  - All episodes have `finalVideoUrl`
  - At least 2 episodes selected
  - Episodes are sorted by sequence order (server-side)
- Process:
  1. Collect `finalVideoUrl` paths in sequence order
  2. Call `assembleVideo()` from `src/lib/video/ffmpeg.ts` with `subtitles: []` and `shotDurations: []` (empty arrays skip subtitle burn-in; the function's `if (subtitles.length > 0)` guard handles this)
  3. The function outputs `{projectId}-concat-{ulid}.mp4` naming — this is acceptable as-is; no need to customize the filename
  4. Optionally update `projects.finalVideoUrl`
- Response: `{ videoUrl: string }` — the API-accessible URL for the merged video

### Storage

- Merged video stored in `uploads/videos/` alongside other videos
- Served via existing `/api/uploads/[...path]` route
- No new DB fields needed; the URL is returned directly to the client

### Download

- Client uses the returned `videoUrl` for both preview (video player) and download
- Download via `<a href={url} download>` or programmatic fetch + blob

---

## Non-Goals

- Markdown syntax-aware parsing (headings as episode breaks, etc.) — treated as plain text
- Auto-selecting "续上集" — always manual opt-in
- Background task queue for merge — sync operation (concat is fast, no re-encoding needed with `-c copy`)
- Handling codec/resolution mismatches between episodes — assumes all episodes use same codec/resolution from generation
- Persistent merge history — merge is on-demand, URL returned in response

## Dependencies

- Existing: `fluent-ffmpeg`, `mammoth`, `unpdf`, `archiver`
- New: none required (markdown is read as plain text via Buffer)
